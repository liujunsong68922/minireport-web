MiniReport是一个报表实现的原型系统，
它的设计目的是在系统设计时作为【原型】来使用，很容易的可以搭建起一个通过Web方式可访问到的数据页面
这个页面具有一定的定制性，核心的展示元素是 线【Line】和标签【Label】,
通过这两者的组合来向客户渲染出一个带有后端数据的页面出来。

系统设计说明如下：
1.为方便在线演示之用，在web.xml中配置H2console，可使用h2console在线查看修改数据库(Mysql)
2.前端页面配置采用XML文件的格式
3.后端采用SpringMVC作为基础架构，以文件流的方式来读取XML文件作为配置项
4.在XML中直接配置数据库访问的SQL语句来作为数据的提取方式，利用JdbcTemplate来直接返回一个List<Map<>>
5.XML定义的UI部分，其报表部分分为报表头【Head】部分和报表体【Detail】部分，
   Detail部分的元素根据返回的数据集合进行重复渲染
6.minireport-web项目基于dtable-web项目演化而来，提供三种渲染方式
  6.1 后台Java Griphics渲染（返回一个图片来进行渲染）
  6.2 前台Canvas渲染（需要H5浏览器支持）
  6.3 前台SVG渲染（需要SVG浏览器支持）
